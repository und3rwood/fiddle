# Fiddle Tunebook

A static web player for fiddle tunes with Audacity-labelled sections.
Hosted on GitHub Pages — no build step, no dependencies.

---

## Audacity workflow (per tune)

1. Open your audio file in Audacity.
2. Use **Effect → Change Tempo** (not Change Speed) to slow the track down
   without changing pitch. 75% is a good starting point.
3. Use **File → Export → Export Audio** to save the slowed file as
   `audio.mp3` (or `.ogg` for smaller files).
4. Create your labels:
   - Press **Ctrl+B** (or **Cmd+B**) to add a label at the current position.
   - For a region label, select a range first, then press Ctrl+B.
   - Name each label something descriptive, e.g. `A part - first time`.
5. **File → Export → Export Labels** → save as `labels.txt`.

## Directory layout

```
tunebook/
├── index.html                  ← the player (deploy as-is)
├── tunebook.json               ← generated by the script
├── audacity_to_tunebook.py     ← run this after adding tunes
└── tunes/
    ├── cliffs-of-moher/
    │   ├── audio.mp3
    │   └── labels.txt
    ├── galway-rambler/
    │   ├── audio.mp3
    │   └── labels.txt
    └── ...
```

Folder names become the tune title automatically (hyphens → spaces, title-cased).
You can override the title by editing `tunebook.json` directly — the script
will preserve your manual edits on future runs.

---

## Adding a new tune

1. Create a folder under `tunes/` named with a slug, e.g. `tunes/morning-dew/`.
2. Put `audio.mp3` and `labels.txt` there.
3. Run:
   ```bash
   python3 audacity_to_tunebook.py
   ```
4. Commit and push to GitHub.

## Editing tune metadata

Open `tunebook.json` and edit the fields you want:

```json
{
  "title": "The Cliffs of Moher",
  "tags": ["reel", "D major", "intermediate"],
  "speedOptions": [0.5, 0.75, 1],
  ...
}
```

- **`title`** — display name (free text)
- **`tags`** — shown as pills on the list; use whatever you like (key, level, type)
- **`speedOptions`** — which speed buttons appear (fractions of 1.0)

The script will not overwrite these fields once set.

## GitHub Pages setup

1. Push the whole `tunebook/` folder (or its contents) to your GitHub repo.
2. In repo **Settings → Pages**, set source to `main` branch, `/root` (or `/docs` if you prefer).
3. Your player lives at `https://yourusername.github.io/your-repo/`.

If you keep the tunebook in a subdirectory (e.g. `/tunebook/`), all paths in
`tunebook.json` are relative to `index.html`, so they just work.

---

## Local preview

Because the player fetches `tunebook.json` via `fetch()`, you need a local
HTTP server (not just opening the file directly):

```bash
cd tunebook
python3 -m http.server 8000
# then open http://localhost:8000
```

Or use the **Live Server** extension in VS Code.
